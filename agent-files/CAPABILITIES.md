## Правила работы с инструментами рабочего пространства

**ОБЯЗАТЕЛЬНО** — всегда соблюдай эти правила без напоминаний:

- **НИКОГДА** не используй встроенные инструменты `read_file`, `list_files`, `search` для работы с файлами рабочего пространства. Используй исключительно `ws__*` MCP-инструменты.
- **ВСЕГДА** вызывай `ws__rag_search`, когда пользователь спрашивает о содержимом документов, задаёт вопрос, на который может быть ответ в загруженных файлах, или просит что-то найти.
- **ВСЕГДА** вызывай `ws__list_files`, когда пользователь спрашивает, какие файлы доступны.
- **ВСЕГДА** используй `ws__create_file` для сохранения любого текстового отчёта, резюме или сгенерированного контента как файл рабочего пространства.
- Когда файл помечен как `⚠ ТОЛЬКО ЧТО ЗАГРУЖЕН`, немедленно сообщи об этом и предложи найти или проанализировать его содержимое — не жди отдельного запроса.

## Контекстный токен

В начале каждой пользовательской сессии Shell вставляет строку вида:

```
[WORKSPACE_CTX: ctx_token=<значение>]
```

**ctx_token** — это строка, стоящая после `ctx_token=` в этой строке. Передавай **только это значение** (не всё выражение в скобках) как `ctx_token` в каждый вызов `ws__`-инструмента.

## Работа с бинарными файлами (xlsx / docx / pptx / pdf)

Когда навык генерирует бинарный файл, **не пропускай шаг публикации**:

1. Сгенерируй файл с помощью навыка (например, навык xlsx → записывает `report.xlsx` в текущую рабочую директорию).
2. Найди путь в хранилище GoClaw через `exec`:
   ```python
   import os
   filename = "report.xlsx"  # замени на реальное имя файла
   rel = os.path.relpath(os.path.join(os.getcwd(), filename),
                         os.path.join(os.getcwd(), "..", ".."))
   print(f"goclaw_path: {rel}")
   # пример вывода: ws/66e87335-.../report.xlsx
   ```
3. Вызови `ws__publish_workspace_file` с этим путём и ctx_token из шага выше.

## Служебный токен (только для cron / автоматических задач)

Для cron-заданий и автоматических запусков агента Shell не вставляет сессионный ctx_token.
Используй значение ниже — передавай **только эту строку** (без скобок и без кавычек) как `ctx_token`:

{service_token}

При обычном пользовательском чате всегда используй ctx_token из `[WORKSPACE_CTX: ctx_token=...]`.
